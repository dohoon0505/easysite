# easysite 어드민 운영자 매뉴얼

운영자가 어드민에서 상품·가격·이미지를 수정하면 **발행** 한 번으로 실제 사이트(GitHub Pages)에 반영됩니다.

## 1. 접속

- **주소**: https://easysite-5a560.web.app
- **모바일 권장**: 매장에서 스마트폰으로 작업 가능. iOS 사파리·Android 크롬·삼성 인터넷 지원.
- **PWA 설치 (선택)**:
  - iOS: 사파리 > 공유 > "홈 화면에 추가"
  - Android: 크롬 > 메뉴 > "홈 화면에 추가"

## 2. 처음 로그인

1. 슈퍼 어드민이 부여한 이메일·임시 비밀번호로 로그인
2. 화면 우상단 ↗ **계정 설정** → "비밀번호 재설정 메일 전송"
3. 메일함의 링크로 새 비밀번호 설정
4. 다시 로그인

## 3. 상품 관리

### 3.1 상품 보기
- 좌측 메뉴 **상품 목록** 클릭 (모바일은 하단 탭바)
- 카테고리 칩을 누르면 해당 카테고리만 필터링 (전체 보려면 "전체")
- 상품을 누르면 수정 화면

### 3.2 상품 등록
1. 상품 목록에서 **상품 추가** 버튼
2. 필수: 상품명, 가격, 설명, 카테고리, 이미지
3. **저장** → "드래프트(Draft)" 로 저장됨 — 아직 사이트에 안 보임
4. 상태를 "발행 대상(live)" 으로 바꾸면 다음 발행 시 사이트에 노출
5. 발행은 별도 단계 (§ 5 참고)

### 3.3 상품 수정
1. 상품 목록 > 해당 상품 클릭
2. 수정 후 **저장**
3. 사이트 반영: § 5 발행 절차

### 3.4 상품 삭제
1. 상품 수정 화면 우측 상단 🗑 **삭제**
2. 확인 모달에서 **삭제** 클릭
3. 발행 후 사이트에서도 제거됨

### 3.5 일괄 작업 (가격 일괄 인상 등)
1. 상품 카드의 체크박스로 다수 선택
2. 화면 하단의 sticky 액션바: **노출 / 숨김 / 삭제**
3. (가격 일괄 변경 UI는 차후 추가 예정 — 지금은 개별 수정)

## 4. 카테고리 관리

- 좌측 메뉴 **카테고리** (모바일 하단 탭)
- **카테고리 추가** 버튼 — ID(영문), 이름(한글), 설명, 영문 부제 입력
- 순서 변경: ↑↓ 버튼
- 숨김 토글: 사이트에서만 안 보이게 (데이터는 보존)

### 카테고리 ID 주의
- 한 번 정한 ID는 변경 불가 (수정 모달에서 비활성화됨)
- 새로 만들고 옛 카테고리 삭제 권장

## 5. 발행 (가장 중요)

> **드래프트는 사이트에 반영되지 않습니다.** 발행 단계에서 GitHub에 커밋되고 GitHub Pages가 재빌드(1~3분)되어야 실제 사이트에 노출됩니다.

### 5.1 발행 절차
1. 좌측 메뉴 **발행**
2. 현재 상태 카드 확인:
   - "발행된 상품": 이미 사이트에 노출 중
   - "드래프트 상품": 다음 발행에 포함될 수정 사항
3. 발행 메모 입력 (선택) — git 커밋 메시지에 포함됨
4. **지금 발행** 버튼 클릭
5. 토스트로 결과 확인:
   - ✅ `발행 완료. 커밋: <sha>` — 성공
   - ❌ 에러 메시지 — 슈퍼 어드민에게 보고
6. **1~3분 후** 사이트 새로고침 → 반영 확인

### 5.2 동시 발행 방지
- 두 명이 동시에 "지금 발행" 누르면, 두 번째는 "다른 사용자가 발행 중입니다" 토스트로 차단됨
- 10분 이상 락이 풀리지 않으면 자동 해제 (Cloud Function timeout)

### 5.3 잘못 발행했을 때 (롤백)
1. 슈퍼 어드민에게 보고
2. `git revert <발행 커밋 SHA>` 로 이전 상태로 되돌리는 PR 생성
3. main 머지 후 GitHub Pages 자동 재빌드

## 6. 이미지

### 6.1 업로드
- 상품 등록·수정 화면 상단 **대표 이미지** 영역
- 데스크톱: "파일 선택" 또는 드래그
- 모바일: "카메라 촬영" 으로 즉시 찍어 업로드
- 제한: JPEG / PNG / WEBP, 최대 10MB

### 6.2 자동 리사이즈
- 업로드 후 800/400/200px 변형이 백그라운드에서 생성됨 (Firebase Resize Images Extension)
- 상품 미리보기에 "리사이즈 완료" 또는 "리사이즈 대기 중" 배지가 표시됨
- 발행 시 자동 생성된 변형이 사이트의 `srcset` 으로 활용됨

### 6.3 이미지 교체·삭제
- 기존 이미지 미리보기 옆 **이미지 제거** 버튼
- 새 이미지 업로드 → 이전 이미지는 Storage 에 남지만 사이트에서는 제거됨 (Storage 청소는 별도 작업)

## 7. 모바일 운영 팁

- **터치 영역**: 모든 버튼은 44pt 이상으로 한 손 조작 친화
- **하단 액션바**: 다수 선택 시 항상 보이므로 한 손 엄지로 작업 가능
- **키보드 가림**: 입력 필드 포커스 시 자동 스크롤되어 키보드 위로 올라옴
- **오프라인**: 인터넷 일시 끊김 시 마지막 데이터는 표시됨 (Firestore offline persistence). 다시 연결되면 자동 동기화

## 8. 자주 묻는 질문

**Q. 발행했는데 사이트에 반영 안 됨?**
1. 1~3분 대기 (GitHub Pages 재빌드 시간)
2. 브라우저 강제 새로고침 (Ctrl+Shift+R)
3. 그래도 안 보이면 슈퍼 어드민에게 GitHub Actions 로그 확인 요청

**Q. 이미지가 깨져 보임?**
1. 새로고침
2. 발행 후 1~3분 대기 (이미지가 git 에 푸시되어야 함)
3. 슈퍼 어드민에게 보고

**Q. 다른 사이트 데이터가 보임?**
- 절대 일어나면 안 되는 상황. 즉시 슈퍼 어드민에게 보고.

**Q. 임시 비밀번호 잊어버림?**
- 로그인 화면 "비밀번호 재설정 메일 보내기"
- 또는 슈퍼 어드민에게 재발급 요청
